# Lab 04 – Deep Dive in Packet Analysis: Using Wireshark and Network Miner

**Course:** CIS 4680 – Introduction to Information Security and Privacy

**Platform:** ACI Learning Skill Labs

## Objective
The objective of this lab is to analyze network traffic using packet analysis tools to identify communication patterns, detect suspicious activity, and understand how data travels across a network. This lab provides hands-on experience with capturing and examining packets, filtering network traffic, and extracting useful forensic information from packet captures.

## Tools & Technologies Used
| Tool | Purpose |
|------|---------|
| Wireshark | Capture and analyze network packets to examine traffic flowing across the network |
| Kali Linux / Windows | Analysis platforms |
| NetworkMiner | Passive network forensic analyzer; file/credential extraction |

## Key Concepts Covered
- **Packet capturing and network traffic analysis**
- **Identifying source and destination IP addresses**
- **Understanding TCP/IP communication**
- **Analyzing common network protocols**
- **Applying packet filters in Wireshark**
- **Detecting suspicious or malicious network activity**

## What I Learned
- Captured and analyzed network traffic using Wireshark.
- Applied display filters to isolate specific packets and protocols.
- Identified source and destination IP addresses, ports, and communication patterns.
- Examined common network protocols such as TCP, UDP, DNS, HTTP, and HTTPS.

## Real-World Application
Packet analysis tools such as Wireshark and NetworkMiner are commonly used by cybersecurity professionals, network administrators, and incident response teams to monitor and troubleshoot network activity. In enterprise environments, these tools help organizations detect malware communication, investigate security incidents, identify unauthorized access attempts, and diagnose network performance issues. For example, a security analyst may use packet analysis during a cyberattack investigation to trace suspicious traffic, identify compromised devices, and determine how attackers moved through the network.
