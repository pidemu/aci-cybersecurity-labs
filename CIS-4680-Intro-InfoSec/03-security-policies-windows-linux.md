# Lab 03 – Implementing Security Policies on Windows and Linux

**Course:** CIS 4680 – Introduction to Information Security and Privacy

**Platform:** ACI Learning – InfoSec Learning Skill Labs

## Objective
Configure and enforce operating system-level security policies on both Windows and Linux platforms. Apply principle of least privilege, set password complexity and account lockout requirements, configure audit logging, and verify that policies are properly applied and enforced across user accounts and system settings.

## Tools & Technologies Used
| Tool | Purpose |
|------|---------|
| Windows Server (Group Policy / Local Security Policy) | Windows policy configuration |
| `passwd`, `chage` | Linux user account management |
| Windows Event Viewer | Verifying audit log generation |

## Key Concepts Covered
- **Windows Group Policy**
- **Password Policy Requirements**
- **Account Lockout Policy** 
- **Principle of Least Privilege** 

## What I Learned
- Security policies are only as effective as their enforcement mechanism. Windows GPO and Linux PAM provide the technical controls, but administrators must verify they are applied correctly.
- Password complexity requirements and account lockout policies work together: complexity raises the cost of offline attacks while lockout limits online attempts.
- The same security objectives (strong passwords, account lockout, logging) require entirely different technical implementations on Windows vs. Linux, highlighting the value of platform-specific knowledge.

## Real-World Application
Security policy configuration is a day-one task for any system administrator hardening a new deployment. Compliance frameworks including PCI-DSS, HIPAA, and NIST SP 800-53 mandate specific password and account controls that map directly to these configurations. SOC analysts rely on audit log data generated by these policies for alert correlation and forensic investigation.
