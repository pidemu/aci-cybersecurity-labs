# Lab 14 – Identifying & Analyzing Network Host Intrusion Detection System Alerts

**Course:** CIS 4680 – Introduction to Information Security and Privacy

**Platform:** ACI Learning Skill Labs

## Objective
The objective of this lab was to identify, monitor, and analyze alerts generated by Network Intrusion Detection Systems (NIDS) and Host Intrusion Detection Systems (HIDS).

## Tools & Technologies Used
| Tool | Purpose |
|------|---------|
| Kali Linux | Attack traffic generation |
| Wireshark | Packet-level correlation of IDS alerts |
| `nmap` | Generating detectable scan patterns |
| Metasploit | Generating exploit traffic for detection |

## Key Concepts Covered
- **IDS vs. IPS**
- **Network IDS (NIDS) vs. Host IDS (HIDS)**
- **False Positives**
  
## What I Learned
- Through this lab, I learned how intrusion detection systems help organizations identify and respond to cybersecurity threats in real time. I gained hands-on experience analyzing IDS alerts, interpreting log data, and distinguishing between normal activity and potential attacks.

## Real-World Application
The skills developed in this lab are commonly used by Security Analysts, SOC Analysts, and Incident Response teams to monitor enterprise environments for cyber threats. Organizations rely on IDS technologies to detect unauthorized activity, investigate security incidents, and strengthen defensive security measures against attacks.
